# 검수 & 발행 체크리스트

> 5관점 검수 + 발행 전 최종 확인을 통합합니다. 자동화 가능 항목과 수동 판단 항목을 구분합니다.

---

## 1. 5관점 검수

초고 완성 후 아래 5관점으로 자가 검수합니다. 모든 항목을 통과해야 발행합니다. 미통과 시 해당 항목을 수정하고 재검수합니다. **최대 2회 재검수 후에도 미통과 시 문제 항목을 명시하고 사람의 판단을 요청합니다.**

### 전략 (블랙 관점) — 수동 판단

- [ ] 이 글이 왜 지금 필요한지 명확한가?
- [ ] "AI 에이전트를 실무에 적용하는 실전 교육" 포지셔닝과 맞는가?
- [ ] 경쟁 콘텐츠 대비 우리만의 관점이 있는가?
- [ ] AI/클로드 관련 기술 설명이 정확한가?

### 논리·팩트 (레드 관점) — 자동 + 수동

**자동화 가능:**
- [ ] 금지 표현 패턴 검출 결과: 0건인가?
- [ ] 모든 수치에 출처가 동반되는가?
- [ ] `<h3>` 태그가 본문 내에 없는가? (`<p><strong>` 패턴 사용)
- [ ] 기술 용어 Lv.2가 첫 등장 시 괄호 설명을 갖추었는가?

**수동 판단:**
- [ ] 논리적 비약 없이 주장이 전개되는가?
- [ ] 수치·기업명·도구명이 정확한가?
- [ ] A/B 등급 소스를 핵심 논거로 사용했는가?

### 실용성 (골드 관점: 35~45세 경영 리더, 비개발자) — 수동 판단

- [ ] "내 사업에 어떻게 쓰라는 건데?"에 답하는가?
- [ ] 기술 용어가 설명 없이 사용되지 않았는가?
- [ ] 비개발자가 따라 할 수 있는가?
- [ ] 읽는 시간 대비 충분히 유용한가?

### 스토리·문체 (퍼플 관점) — 자동 + 수동

**자동화 가능:**
- [ ] 60자 초과 문장이 없는가?
- [ ] 3줄 요약이 입니다/합니다 체인가?
- [ ] em dash(—) 사용이 없는가?
- [ ] 컨텍스트 섹션이 5문장 이하인가?
- [ ] 모든 문단이 3~5문장인가? (초과 시 분리)
- [ ] 참고자료가 APA 형식인가? (연도+월, [Video/Article], URL 전문)
- [ ] Roasting quote가 4곳 동기화되었는가? (blockquote, data 속성, JS 상수, posts-index.json)
- [ ] 비즈니스 비용 섹션이 빈 "[해당 없음]"으로 존재하지 않는가? (없으면 생략)

**수동 판단:**
- [ ] 첫 3문장이 독자를 붙잡는가?
- [ ] 각 문단의 첫 문장이 핵심을 대표하는가? (Topic Sentence)
- [ ] 끝까지 읽히는 구조인가?

### UI·성능 (화이트 관점) — 자동 + 수동

**자동화 가능:**
- [ ] lazy loading 적용 (히어로 제외)
- [ ] word-break: keep-all 적용

**수동 판단:**
- [ ] 시맨틱 태그, id/class 네이밍, 섹션 구조가 올바른가?
- [ ] 모바일(375px)~데스크톱(1440px)에서 깨지는 요소가 없는가?
- [ ] alt 텍스트, aria-label이 적용되었는가?

### 의견 충돌 시 우선순위

**팩트 > 논리 > 실용성 > 전략 > 표현**

## 2. 자동 검수 패턴

스크립트 또는 스킬에서 자동 실행할 수 있는 검증 항목입니다.

```
# 금지 표현 검출 (regex 패턴 매칭)
혁명적|획기적|할 수밖에|모든 기업이|놀라운|충격적인|패러다임 시프트|게임 체인저|폭발적 성장|누구나 쉽게|—

# 문장 길이 (마침표·물음표·느낌표 기준으로 분리 후, 공백 제외 글자 수가 60자 초과인 문장 검출)

# 종결어미 (3줄 요약 섹션 내 문장 말미가 입니다/합니다/됩니다로 끝나는지 확인)

# em dash 검출 (문자 '—' U+2014 존재 여부)

# h3 태그 검출 (본문 내 <h3> 존재 여부, 있으면 <p><strong>으로 교체)

# 컨텍스트 문장 수 (section-context 내 문장이 5개 초과인지 검출)

# 문단 길이 (하나의 <p> 태그 내 문장이 6개 이상인 경우 분리 필요)

# Roasting 동기화 (blockquote, data-roasting-quote, ROASTING_QUOTE, posts-index.json 4곳 일치 확인)

# 참고자료 형식 ("영상 보기" 같은 축약 링크 텍스트 검출, URL 전문으로 교체)

# 빈 비즈니스 비용 섹션 ([비즈니스 비용: 해당 없음] 텍스트 검출, 있으면 섹션 삭제)

# lazy loading 누락 (<img 태그 중 히어로 이미지 제외, loading="lazy" 미적용 검출)

# 접속 부사 연속 (또한|그러나|하지만|따라서|그래서|한편|반면 으로 시작하는 문장이 2개 연속인 경우)
```

## 3. 발행 전 최종 확인

### 작성 전

- [ ] 카테고리는? (리서치 / 리더 / 기업 / 기술 / 생존)
- [ ] 핵심 메시지 1문장은?
- [ ] 독자가 읽고 나서 할 행동은?

### 퇴고

- [ ] 최우선 3원칙 통과 (자연스러운 한국어, 주술 근접, 이해하기 쉽게)
- [ ] 금지 표현 패턴 검출 0건
- [ ] 3줄 요약 입니다/합니다 체
- [ ] 60자 초과 문장 없음
- [ ] 출처 없는 수치 없음
- [ ] em dash 없음

### 발행 직전

- [ ] 5관점 검수 완료
- [ ] 오탈자·맞춤법·링크 확인
- [ ] 모바일 미리보기 (단어 중간 줄바꿈 없는지)
- [ ] HTML 파일 저장 ({category}/YYYY-MM-DD-slug.html)
- [ ] 이미지 어셋 저장 완료
- [ ] lazy loading 적용 (히어로 제외)
- [ ] 읽기 진행률 바·TOC·프롬프트 복사 동작 확인
- [ ] 모바일 공유 바 본문 가림 없음
- [ ] _posts/ 마크다운 파일 저장
- [ ] 메타 디스크립션 + OG 태그 + JSON-LD 작성
- [ ] word-break: keep-all 적용

### JS 데이터 업데이트

- [ ] posts-index.json에 새 포스트 추가 (배열 최상단, 최신순)
- [ ] 이전 최신 글의 next_post 필드 업데이트
- [ ] assets/js/posts-data.js 동기화 (node sync-posts.js 또는 직접 수정)
- [ ] 동기화 검증: posts-index.json과 posts-data.js의 포스트 수 일치 확인
- [ ] 홈 페이지에서 새 글 목록 표시 확인

> **주의**: posts-index.json이 단일 소스입니다. posts-data.js를 직접 수정한 경우, 반드시 posts-index.json에도 동일 내용을 반영하세요. 불일치 시 검색·필터가 깨집니다.
