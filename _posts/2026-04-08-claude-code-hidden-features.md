## 핵심 인사이트

**첫째, 병렬 도구 호출이 처리 속도를 결정합니다.** "동시에 파일 A와 B를 읽으세요"라고 명시하면 순차 처리보다 빠릅니다. 신호가 없으면 기본값으로 순차 처리합니다(MindStudio, 2026).

**둘째, CLAUDE.md가 프로젝트의 장기 기억입니다.** 프로젝트 루트의 CLAUDE.md 파일이 세션 간 지속되는 기억 레이어입니다. 핵심 규칙 10개 이내로 유지하면 컨텍스트 낭비 없이 일관성을 확보합니다.

**셋째, 에이전트 루프에는 종료 조건이 필수입니다.** 명확한 완료 상태를 정의하지 않으면 루프에 갇힙니다. "로그에서 오류를 찾으면 설명하고, 없으면 알려주세요"처럼 조건을 명시해야 합니다(MindStudio, 2026).

**넷째, 확장 사고를 의도적으로 유발할 수 있습니다.** "이 아키텍처 결정의 장단점을 분석해 주세요"처럼 분석 작업으로 표현하면 더 깊은 추론이 나옵니다.

## 8가지 작동 원리

1. 병렬 도구 호출 - 독립 작업을 동시 실행 (명시 필요)
2. CLAUDE.md 기억 - 세션 간 지속 기억 레이어
3. 불필요한 사과 제거 - 직접적 프롬프트가 더 효율적
4. Bash 우선 사용 - 파일/Git/패키지는 Bash로
5. 차등 파일 편집 - 전체 재작성 대신 정확한 문자열 매칭
6. 루프 종료 조건 - 명확한 완료 상태 정의 필수
7. Git 안전 지침 - 명시 없으면 파괴적 작업 회피
8. 확장 사고 유발 - 분석 작업 표현이 깊은 추론 유도

## 내일 당장 할 수 있는 것

1. **CLAUDE.md를 작성합니다.** 현재 프로젝트 기술 스택, 코딩 컨벤션, 금지 패턴을 30분 안에 문서화합니다. (30분)
2. **병렬 요청을 실습합니다.** 독립적인 작업 2개를 하나의 요청으로 묶습니다. (즉시 적용)
3. **루프 종료 조건을 명시합니다.** "완료 조건: ~" 구조를 반복 작업에 추가합니다. (10분)
