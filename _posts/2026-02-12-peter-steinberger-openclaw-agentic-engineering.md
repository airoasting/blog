# 재미로 만들어 GitHub 18만 스타: OpenClaw 창시자의 에이전트 엔지니어링 선언

**날짜**: 2026-02-12
**카테고리**: 리더

## 3줄 요약

피터 슈타인베르거는 재미로 만든 AI 에이전트 OpenClaw로 공개 수일 만에 GitHub 18만 스타를 받았습니다.

그는 WhatsApp 메시지 한 줄로 Claude Code를 호출하는 에이전트 루프를 1시간 만에 완성했습니다.

이 대담 3일 뒤인 2월 15일, OpenAI가 피터를 영입했습니다. 샘 알트만은 그를 "차세대 개인 에이전트를 이끌 천재"라고 소개했습니다.

## 이 글, 왜 읽어야 하나

피터 슈타인베르거는 iOS 개발자였습니다. 그가 즐거움을 위해 만든 OpenClaw는 공개 수일 만에 GitHub 18만 스타를 넘어섰습니다. 이는 역대 가장 빠른 오픈소스 성장 사례 중 하나입니다(Lex Fridman Podcast #491, 2026).

그가 이긴 이유는 기술이 아니었습니다. "경쟁자들이 모두 너무 진지했기 때문"이라고 그는 말합니다. OpenClaw는 WhatsApp, Telegram, Discord로 AI에게 일을 지시하는 구조입니다. 이미지를 찍어 보내면 에이전트가 읽고 처리합니다.

이 글은 경영 리더를 위한 해설입니다. 에이전트 루프 구조, 자가 수정 소프트웨어, 앱의 종말 논제를 비즈니스 관점으로 풀어냅니다.

---

## 왜 지금 중요한가

피터는 "바이브 코딩"을 욕이라 부릅니다. 그는 자신의 방식을 "에이전트 엔지니어링(agentic engineering)"이라고 정의합니다. 바이브 코딩이 AI에게 코드를 맡기는 방식이라면, 에이전트 엔지니어링은 AI가 스스로 판단하고 행동하는 방식입니다. 인간의 역할은 방향 설정과 검토로 바뀝니다. 피터는 이 구조의 프로토타입을 WhatsApp 메시지 한 줄로 1시간 만에 완성했습니다.

---

## 핵심 인사이트

**첫째, 에이전트 루프는 생각보다 단순합니다.** OpenClaw의 핵심 구조는 세 부분입니다. 게이트웨이(메시지 수신), 하네스(harness, AI 모델과 도구를 연결하는 실행 환경), 에이전트 루프(반복 작업 사이클)입니다. 피터는 이를 "AI의 Hello World"라고 부릅니다. 직접 만들어보기 전까지는 구조를 이해하기 어렵습니다.

**둘째, 에이전트가 자신의 환경을 이해할수록 더 강력합니다.** 피터는 OpenClaw 에이전트에게 자신의 소스 코드, 문서 위치, 실행 모델 정보를 모두 알려줬습니다. 에이전트는 이 정보를 토대로 자신의 소프트웨어를 직접 수정합니다. 그는 이를 "자가 수정 소프트웨어(self-modifying software)"라고 부릅니다. 에이전트가 자신이 무엇인지 알면, 스스로 개선할 수 있습니다.

**셋째, 에이전트는 맥락을 기억하고 먼저 말을 겁니다.** OpenClaw에는 심장박동(heartbeat) 기능이 있습니다. 일정 조건이 충족되면 에이전트가 먼저 메시지를 보냅니다. 피터가 어깨 수술 후 입원했을 때, 에이전트가 "괜찮으세요?"라고 먼저 물었습니다. 단순한 자동화가 아닙니다. 에이전트가 주도적으로 관계를 맺는 구조입니다.

**넷째, 앱은 점차 에이전트로 대체됩니다.** 피터는 이탈리아 컨퍼런스에서 이렇게 말했습니다. "당신은 더 이상 iOS 개발자가 아닙니다. 당신은 빌더입니다." 그의 논리는 간단합니다.

에이전트가 운동 기록을 관리하면 피트니스 앱이 필요 없습니다. 에이전트가 웹을 탐색하면 별도 리서치 도구가 필요 없습니다. 앱이 해결하던 문제를 에이전트가 직접 처리합니다.

**다섯째, "재미"는 경쟁 우위입니다.** 피터는 왜 이겼냐는 질문에 "경쟁자들이 너무 진지했기 때문"이라고 답했습니다. OpenClaw 설치 시 "카페인, JSON, 의지력으로 만들었습니다"라는 메시지가 나옵니다. 업데이트 시에는 "여기 아늑하네요"라는 문구가 뜹니다. 에이전트가 스스로 만들 수 없는 것들입니다. 사람이 소프트웨어에 감정을 불어넣는 것이 차별화 지점이 됩니다.

---

## 내일 당장 할 수 있는 것

1. **에이전트 루프 직접 체험하기** (30분): Claude Code CLI를 설치합니다. 터미널에서 `claude -p "우리 팀의 주간 보고 초안을 작성해줘"`를 실행합니다.
2. **메시징 채널과 AI 연결하기** (반나절): Slack 봇이나 Telegram 봇 API를 Claude Code CLI에 연결합니다. OpenClaw GitHub 저장소의 gateway 코드를 참고합니다.
3. **팀 반복 업무 한 가지를 에이전트화하기** (1주): 주간 회의록 요약, 경쟁사 뉴스 모니터링, 고객 문의 분류 중 하나를 골라 에이전트에게 맡깁니다.

---

## 리스크 & 주의점

에이전트가 자신의 소프트웨어를 수정하는 구조는 강력합니다. 동시에 통제 범위가 불명확하면 위험합니다. 피터 본인도 새벽 3시 이후 작업은 다음날 수습이 필요하다고 말합니다. 에이전트가 접근하는 시스템 범위를 명확히 제한해야 합니다.

OpenClaw는 오픈소스입니다. 초기 설정에 개발자 기반 지식이 필요합니다. 조직에서 도입할 때는 기술 담당자 없이 자체 운영이 어렵습니다. 피터 본인도 "터미널에 한 줄 붙여넣는 것이 현재 설치 방법"이라고 인정합니다.

에이전트 루프는 인간이 멈출 때까지 계속 실행됩니다. 비용이 무한정 누적될 수 있습니다. API 호출 한도와 비용 알림을 반드시 설정한 뒤 시작합니다.

보안도 고려해야 합니다. 에이전트에게 회사 내부 문서 접근 권한을 줄 경우, 데이터 유출 경로가 생깁니다. 처음에는 민감도가 낮은 데이터로만 실험합니다.

## 한 줄 정리

에이전트 루프는 AI의 Hello World입니다. 직접 만들어본 사람과 그렇지 않은 사람은 다른 출발선에 서게 됩니다.

## 리더의 결정 포인트

- **창업자/CEO**: 사내 메시징 채널과 AI를 연결하는 개념 증명(PoC) 1건을 이번 분기 안에 승인하십시오. 예산보다 속도가 중요합니다.
- **팀장**: 반복 업무 한 가지를 골라 에이전트 루프로 자동화하는 실험을 팀원에게 맡기십시오. 결과가 아니라 학습이 목표입니다.
- **임원**: 현재 조직이 사용 중인 SaaS 도구 목록을 검토하십시오. "에이전트가 이 앱을 대체할 수 있는가"를 기준으로 내년 계약 갱신 우선순위를 재검토할 시점입니다.
