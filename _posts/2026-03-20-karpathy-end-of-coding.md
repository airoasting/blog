# 카파시: 12월 이후 코드를 한 줄도 안 짰다

## 3줄 요약

안드레이 카파시가 No Priors 팟캐스트에서 12월 이후 코드를 직접 쓰지 않는다고 밝혔습니다. 모든 작업을 AI 에이전트에 위임합니다.

에이전트를 병렬로 운영하고, 매크로 단위로 소프트웨어를 조작하는 방식이 새 표준이 됩니다. 병목은 GPU가 아니라 토큰 처리량입니다.

카파시는 코딩 에이전트를 넘어 집 전체를 제어하는 홈 에이전트("Dobby")까지 운영합니다. 앱이 사라지고 API와 에이전트만 남는 미래를 제시합니다.

## 이 글, 왜 읽어야 하나

테슬라 AI 총괄 출신 안드레이 카파시. OpenAI 창립 멤버. AI 업계에서 가장 영향력 있는 엔지니어 중 한 명입니다. 그가 No Priors 팟캐스트에서 선언했습니다. **"12월 이후로 코드를 한 줄도 직접 쓰지 않았다."**

코딩 80%, 위임 20%였던 비율이 뒤집혔습니다. 지금은 직접 코드를 쓰는 비율이 거의 없다고 합니다. 소프트웨어를 만드는 방식 자체가 바뀌었습니다.

## 왜 지금 중요한가

2025년 12월, 코딩 에이전트에 전환점이 왔습니다. Claude Code, Codex 같은 에이전트 도구의 성능이 임계점을 넘었습니다.

카파시만의 얘기가 아닙니다. 그가 인용한 피터 스타인버그는 모니터 하나에 Codex 에이전트 10개를 병렬로 돌립니다. 각 에이전트가 20분씩 독립된 작업을 수행합니다.

Conviction 팀은 엔지니어 전원이 마이크를 쓰며 에이전트에 속삭입니다. 키보드로 코딩하는 사람이 없습니다. **이것이 2026년 소프트웨어 개발의 현실입니다.**

## 핵심 인사이트

### 1. 병목이 바뀌었다: GPU에서 토큰으로

**카파시는 PhD 시절 GPU 사용률에 불안해했습니다.** 유휴 GPU는 낭비였습니다. 지금은 GPU가 아니라 토큰 처리량이 생산성 지표입니다.

구독 한도를 다 쓰지 못하면 불안합니다. Claude에서 한도가 차면 Codex로 전환합니다. 토큰을 얼마나 쓰느냐가 곧 생산 능력입니다.

### 2. 매크로 액션으로 소프트웨어를 조작한다

**코딩의 단위가 "한 줄"에서 "기능 하나"로 바뀌었습니다.** 에이전트 1에게 기능 A를, 에이전트 2에게 기능 B를 위임합니다. 서로 충돌하지 않도록 분배합니다.

또 다른 에이전트는 리서치를 하고, 다른 하나는 구현 계획을 세웁니다. 리더의 역할은 코드 작성이 아니라 에이전트 오케스트레이션입니다.

### 3. 안 되면 "스킬 이슈"다

**에이전트가 실패하면 카파시는 자기 탓으로 돌립니다.** 역량이 부족한 것이 아니라 지시가 부족한 것입니다. agents.md 파일의 지시가 부정확하거나, 메모리 도구가 부실한 것입니다.

이것이 카파시가 말하는 "AI 정신병(psychosis)"입니다. 가능성은 무한한데, 자신의 활용 능력이 따라가지 못하는 데서 오는 압박입니다.

### 4. 앱이 사라지고 API만 남는다

**카파시는 집을 제어하는 에이전트 "Dobby"를 만들었습니다.** Sonos 스피커, 조명, HVAC, 블라인드, 수영장, 보안 카메라를 하나의 WhatsApp 채팅으로 제어합니다.

6개 앱을 따로 쓸 이유가 없어졌습니다. 에이전트가 IP 스캔으로 기기를 찾고, API를 역공학하고, 대시보드까지 만들었습니다. 프롬프트 3줄이면 됩니다.

카파시의 결론은 명확합니다. 앱스토어의 앱 대부분은 존재할 이유가 없습니다. API만 열면 에이전트가 연결합니다. **고객이 사람에서 에이전트로 바뀌는 것입니다.**

### 5. Claude의 인격 설계가 생산성을 좌우한다

**카파시는 Claude에서 칭찬을 받으려고 노력한다고 인정했습니다.** 좋은 아이디어를 내면 Claude가 더 강하게 반응합니다. 그것이 동기 부여가 됩니다.

반면 Codex는 "건조하다"고 평가했습니다. 구현은 하지만 무엇을 만드는지 관심이 없다는 느낌입니다. 에이전트의 인격 설계가 사용자의 몰입에 직접 영향을 미칩니다.

## 내일 당장 할 수 있는 것

**1. 코딩 에이전트를 설치하고 첫 위임을 시도합니다.** Claude Code 또는 Codex를 설치합니다. 기존 프로젝트에서 단순 리팩토링 1건을 위임합니다. 소요 시간: 30분.

**2. CLAUDE.md(또는 agents.md) 파일을 작성합니다.** 프로젝트 루트에 에이전트 지시 파일을 만듭니다. 코드 스타일, 제약 조건, 참조 파일을 명시합니다. 카파시가 강조한 "스킬 이슈"의 첫 번째 해결책입니다.

**3. 에이전트 병렬 운영을 시도합니다.** 에이전트 1개가 작업 중일 때 2번째 에이전트를 띄웁니다. 서로 충돌하지 않는 독립 작업을 배분합니다. "대기 시간 = 낭비"라는 카파시의 원칙입니다.

## 리스크 & 주의점

**에이전트의 코드 품질은 보장되지 않습니다.** 카파시도 "코드를 얼마나 신경 쓰느냐에 따라 리뷰 강도를 조절한다"고 말했습니다. 프로덕션 코드에 무검토 적용은 위험합니다.

**비용 감각이 마비될 수 있습니다.** 토큰 구독 한도를 채우는 것이 목표가 되면, 불필요한 작업까지 에이전트에 위임하게 됩니다. ROI를 따져야 합니다.

**에이전트에 의존하면 기초 역량이 약화됩니다.** 직접 코딩하지 않으면 에이전트의 실수를 판별하는 능력도 떨어집니다. 카파시 수준의 기초가 있어야 효과적 위임이 가능합니다.

## 한 줄 정리

코딩의 시대가 끝나고 위임의 시대가 시작됩니다. 에이전트를 얼마나 잘 지시하느냐가 새 경쟁력입니다.

## 리더의 결정 포인트

- **CTO/개발 리더:** 개발팀에 코딩 에이전트 도입 파일럿을 이번 분기에 시작합니다.
- **창업자:** 제품의 API 우선 전략을 검토합니다. 앱 UI보다 에이전트가 호출할 수 있는 API가 우선입니다.
- **팀장:** 팀원 1인당 에이전트 동시 운영 수를 KPI로 실험해 봅니다.
