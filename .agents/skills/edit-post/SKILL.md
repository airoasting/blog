# 포스트 퇴고 & 수정

기존 포스트 파일을 읽고, `guides/editorial-rules.md`의 규칙에 따라 퇴고합니다.

## 입력

`$ARGUMENTS`에서 포스트 파일 경로를 받습니다.

예시:
- `/edit-post tech/2026-03-10-openclaw-rl.html`
- `/edit-post research/2026-03-05-brain-fry.html`

## 실행 단계

### Step 1: 포스트 읽기

지정된 HTML 파일을 읽고 본문 콘텐츠를 추출합니다.

### Step 2: 가이드 로드

`guides/editorial-rules.md`를 읽어 편집 규칙을 컨텍스트에 로드합니다.

### Step 3: 최우선 3원칙 검수

**다른 검수보다 먼저** 최우선 3원칙 위반을 검출하고 수정합니다. 이 단계에서 전체 본문을 문장 단위로 점검합니다.

#### ⚠️ 최우선 3원칙 (위반 시 즉시 수정)

**원칙 1. 자연스러운 한국어로 쓴다.**
- 번역체 검출: "~라고 할 수 있겠습니다", "~라는 점에서", "~에 대해" 남용, "~것으로 보입니다", "~되어지다"
- 조사 오용: "~에 대해서는", "~로부터의"
- 소리 내어 읽었을 때 어색한 문장 → 자연스러운 한국어로 재작성

**원칙 2. 주어와 술어를 가깝게 놓는다.**
- 주술 거리 검출: 주어와 술어 사이에 수식어가 3개 이상 끼인 문장
- 나쁜 예: "AI 에이전트는 반복 업무를 자동화하고 내부 문서를 검색하며 고객 응대까지 처리하는 기능을 통해 기업의 운영 효율성을 높이는 역할을 합니다."
- 수정 방법: 문장을 나누어 주어와 술어를 근접시킨다

**원칙 3. 이해하기 쉽게 쓴다.**
- 한 문장에 정보 2개 이상 → 분리
- 앞 문장 없이 뒷 문장이 이해 안 되는 경우 → 독립적으로 재작성
- 추상적 표현("생산성 향상", "효율 개선") → 구체적 수치/행동으로 교체

> **3원칙 위반은 0건이 될 때까지 반복 수정합니다.** 이 단계를 통과해야 Step 4로 넘어갑니다.

### Step 4: 자동 검수

퇴고 단계 검수입니다. `/publish-post`에서 최종 재확인합니다. 아래 항목을 자동으로 검출합니다:

**금지 표현 검출:**
```
혁명적|획기적|할 수밖에|모든 기업이|놀라운|충격적인|패러다임 시프트|게임 체인저|폭발적 성장|누구나 쉽게|—
```

**문장 길이 체크:**
- 60자 초과 문장 목록 추출

**종결어미 체크:**
- 3줄 요약 섹션이 입니다/합니다 체인지 확인

**em dash 검출:**
- — 문자 사용 여부

**`<h3>` 태그 검출:**
- 본문 내 `<h3>` 존재 시 `<p><strong>` 패턴으로 교체

**문단 길이 체크:**
- 하나의 `<p>` 태그 내 문장이 6개 이상인 경우 분리 필요

**컨텍스트 문장 수 체크:**
- "왜 지금 중요한가" 섹션이 5문장을 초과하는 경우

**기술 용어 Lv.2 체크:**
- 첫 등장 시 괄호 설명이 없는 전문 용어 검출

**참고자료 형식 체크:**
- "영상 보기 ↗" 같은 축약 링크 텍스트 → URL 전문으로 교체
- APA 형식: `저자. (연도, 월). *제목* [Video/Article]. 플랫폼. URL`

**빈 비즈니스 비용 섹션 검출:**
- "[비즈니스 비용: 해당 없음]" 텍스트 존재 시 섹션 삭제

**Roasting quote 동기화 체크:**
- blockquote, data-roasting-quote, JS ROASTING_QUOTE 3곳 일치 확인
- 종결어미가 높임말(~입니다/~합니다/~습니까?)인지 확인. ~다/~나?/~인가? 형태 발견 시 즉시 변환

**접속 부사 연속 사용:**
- 연속 2문장 이상 접속 부사로 시작하는 경우

### Step 5: 검수 리포트

자동 검수 결과를 리포트합니다:

```
## 자동 검수 결과

### 금지 표현: {N}건
- 줄 {N}: "{문장}" → "{대체 제안}"

### 60자 초과 문장: {N}건
- 줄 {N}: "{문장}" ({M}자) → 분리 제안

### 종결어미 불일치: {N}건
### em dash: {N}건
### 접속 부사 연속: {N}건
```

### Step 6: 자동 수정

검출된 모든 문제를 **즉시 자동으로 수정 적용**합니다 (사용자 승인 대기 없음):

1. 금지 표현 → 대체 표현으로 교체
2. 60자 초과 → 문장 분리
3. 종결어미 → 입니다/합니다 체 변환
4. em dash → 쉼표, 마침표, 괄호 대체
5. 접속 부사 연속 → 접속 부사 제거 또는 문장 재구성
6. `<h3>` → `<p><strong>` 패턴으로 교체
7. 참고자료 축약 링크 → URL 전문 APA 형식

### Step 7: 완료 보고

수정 전후 비교 요약을 제시합니다:
- 총 수정 건수
- 주요 변경 사항
- 수동 검토 참고 사항 (번역체 의심 문장, Topic Sentence 불명확 문단 등)
